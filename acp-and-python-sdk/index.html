    ACP 与 Python SDK：让编辑器与 Agent 说同一种语言¶

    TL;DR¶

    • Agent Client Protocol（ACP）把“编辑器 ↔ Agent”的交互标准化，像 LSP 之于语言服务一样，解耦能力提供方与承载方。
    • 我写了一个 Python SDK（agent-client-protocol），用最小心智负担把你的 Python Agent 接到 Zed 等 ACP 客户端上，走 stdio + JSON-RPC。
    • 一条命令安装；几十行代码跑通；支持在 Zed 中作为外部 Agent 运行；内置一个可跑的 Mini SWE Agent 桥接示例。

    参考： - 官方站点与协议文档：https://agentclientprotocol.com - Zed 外部 Agent 集成指南：https://zed.dev/docs/ai/external-agents - Python SDK 文档与示例：https://psiace.github.io/agent-client-protocol-python

    为什么写这篇¶

    我不太想再讨论“画一个复杂的 Agent 架构图”了。我更关心两件事：

    • 让系统把话说清楚（协议与交互尽量标准、可观测）；
    • 让事情先跑起来（最小路径把反馈回路接通）。

    ACP 正好提供了“说清楚”的契约；而 Python SDK 则提供了“先跑起来”的路径。本文把两者合在一起说，顺便交付一个你可以立刻跑的最小例子与在 Zed 中接入的方法。

    ACP 是什么，为什么需要 ACP¶

    官方介绍： - Introduction：https://agentclientprotocol.com/overview/introduction - Architecture：https://agentclientprotocol.com/overview/architecture

    一句话版：ACP 是编辑器与 Agent 之间的标准对话方式。没有它，每个编辑器都要为每个 Agent 单独写适配；每个 Agent 也要学会不同编辑器的私有 API。后果是：

    • 集成成本高：组合数爆炸；
    • 兼容性差：Agent 只能在少数编辑器里跑；
    • 锁定效应强：选了哪个 Agent，往往就被它支持的界面形态绑死了。

    ACP 把这层对话协议抽象出来，类似当年的 LSP：实现了 ACP 的 Agent，在任何支持 ACP 的编辑器里都能工作；支持 ACP 的编辑器也能一次性获得整个生态的 Agent。

    设计要点（把“话”说清楚）¶

    • 传输：JSON-RPC over stdio。简单、可移植、可观测。
    • 内容：沿用 MCP 的 JSON 表达（能复用就复用），也定义了 Agent 场景特有的类型（比如 diff 展示）。
    • 文本：默认是 Markdown，足够表达富文本，但对编辑器渲染能力的要求不过分。
    • 能力：Prompt、工具调用、文件系统访问、Agent 计划等都在协议里有第一类抽象。
    • 支持矩阵（文档中会不断更新）：
    • 编辑器侧：Zed、Neovim（CodeCompanion / avante.nvim）等
    • Agent 侧：Gemini CLI、Claude Code（适配器）等

    参考： - Protocol 总览：https://agentclientprotocol.com/protocol/overview - Zed 外部 Agent 集成文档：https://zed.dev/docs/ai/external-agents

    为什么做一个 Python SDK¶

    Python 在 Agent 生态里是事实标准：工具链丰富、模型 SDK 广、跑脚本部署容易。协议有了，但如果没有“开箱即用的 Python 端实现”，工程起步成本依然高。这个 SDK 试图做到：

    • 安装简单：pip 装完就能用；
    • API 收敛：暴露 AgentSideConnection / ClientSideConnection 两个常见入口；
    • 最小心智负担：几十行代码接入、先把反馈回路跑起来；
    • 演示即文档：Mini SWE Agent 桥接例子，把流式输出、工具调用、最终提交这些典型交互串起来。

    最小可用示例¶

    下面是一个最小 Echo Agent：初始化握手后，在每次 prompt 中发送一段流式文本，并结束回合。

    import asyncio
from acp import (
    Agent, AgentSideConnection, Client,
    InitializeRequest, InitializeResponse,
    PromptRequest, PromptResponse,
    SessionNotification, stdio_streams, PROTOCOL_VERSION
)
from acp.schema import ContentBlock1, SessionUpdate2

class EchoAgent(Agent):
    def __init__(self, client: Client):
        self.client = client
    async def initialize(self, _p: InitializeRequest) -> InitializeResponse:
        return InitializeResponse(protocolVersion=PROTOCOL_VERSION)
    async def prompt(self, p: PromptRequest) -> PromptResponse:
        await self.client.sessionUpdate(SessionNotification(
            sessionId=p.sessionId,
            update=SessionUpdate2(
                sessionUpdate="agent_message_chunk",
                content=ContentBlock1(type="text", text="Hello from ACP (Python)")
            )
        ))
        return PromptResponse(stopReason="end_turn")

async def main() -> None:
    reader, writer = await stdio_streams()
    AgentSideConnection(lambda c: EchoAgent(c), writer, reader)
    await asyncio.Event().wait()

if __name__ == "__main__":
    asyncio.run(main())

    安装：

    pip install agent-client-protocol

    在 Zed 中运行¶

    在 Zed 的 settings.json 里增加一个自定义 Agent 服务（完整说明见官方文档：https://zed.dev/docs/ai/external-agents）：

    {
  "agent_servers": {
    "Echo Agent (Python)": {
      "command": "/abs/path/to/python",
      "args": ["/abs/path/to/your_echo_agent.py"],
      "env": {}
    }
  }
}

    打开 Zed 的 Agents 面板，选择 “Echo Agent (Python)” 即可启动一条会话。Mini SWE Agent 的桥接示例也可以用同样方式运行（见下方链接）。

    和“传统集成”的不同¶

    • 一致性：编辑器与 Agent 的交互语义在协议中被精确定义，减少了“靠猜”的部分。
    • 可替换性：Agent 与编辑器可以独立演进、自由组合。
    • 可观测性：基于 JSON-RPC over stdio，抓包 / 日志 / 重放都很容易；Zed 也内置了 ACP 调试视图。

    我的做事原则¶

    • 先跑起来，再优化：先把“请求 → 响应 → 流式更新 → 结束”这条回路打通；
    • 简单优先：优先选择本地守护（stdio）与最小依赖；
    • 可观测优先：日志与事件尽可能透明，方便复盘和定位；
    • 可替换：模型、后端、工具执行器都不固化；协议稳定、实现可换。

    尾声¶

    我欣赏“能把事情说清楚、也能把事情做成”的工程实践。ACP 让我们在“说清楚”这件事上迈了一步；Python SDK 则让“做成”更轻一点。欢迎你把自己的 Agent 先接起来，跑一遍，看看它在编辑器里的真实表现，再谈优化与花活。

    如果你已经有一个 Python 项目的 Agent 内核，通常只需要把入口包上 AgentSideConnection，几分钟就能跑进 Zed。剩下的，就交给反馈回路与观察。
